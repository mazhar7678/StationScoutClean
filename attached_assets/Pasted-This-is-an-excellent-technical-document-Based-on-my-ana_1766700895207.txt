## Root Cause Analysis

### Most Likely Cause: GestureHandlerRootView Missing or Misconfigured

Looking at your architecture, I see you're using:
- `react-native-gesture-handler` ~2.28.0
- React Native Paper (which depends on gesture handler)
- React Navigation (which also uses gesture handler)

**The #1 cause of "button presses not registering" on Android is improper gesture handler initialization.**

---

## Immediate Fixes to Try

### Fix 1: Verify index.js Import Order

Your `index.js` **must** have this exact order:

```javascript
// index.js

// 1. Gesture handler MUST be first (before React, before anything)
import 'react-native-gesture-handler';

// 2. Your Hermes patches second
import './src/patches/lokijs-hermes-patch';

// 3. Then everything else
import { registerRootComponent } from 'expo';
import App from './App';

registerRootComponent(App);
```

**Why this matters:** On Android, `react-native-gesture-handler` replaces the native gesture system. If it's imported after other React Native code has initialized, touch events get lost.

---

### Fix 2: Wrap Entire App in GestureHandlerRootView

Your `App.tsx` must have `GestureHandlerRootView` as the **outermost** wrapper:

```typescript
// App.tsx

import 'react-native-gesture-handler'; // Also import here for safety
import { GestureHandlerRootView } from 'react-native-gesture-handler';
import { SafeAreaProvider } from 'react-native-safe-area-context';
import { PaperProvider } from 'react-native-paper';
import { NavigationContainer } from '@react-navigation/native';
// ... other imports

export default function App() {
  return (
    // GestureHandlerRootView MUST be outermost
    <GestureHandlerRootView style={{ flex: 1 }}>
      <SafeAreaProvider>
        <PaperProvider theme={theme}>
          <NavigationContainer>
            {/* Your app content */}
          </NavigationContainer>
        </PaperProvider>
      </SafeAreaProvider>
    </GestureHandlerRootView>
  );
}
```

**Critical:** The `style={{ flex: 1 }}` is required! Without it, the gesture handler view has zero height and won't capture touches.

### Fix 3: Check for Conflicting Touch Handlers in LoginScreen

Based on your document, the LoginScreen likely has a structure like this. Look for these issues:

```typescript
// PROBLEMATIC PATTERNS TO CHECK FOR:

// ❌ ScrollView with keyboard dismiss might intercept touches
<ScrollView 
  keyboardShouldPersistTaps="never"  // Change to "handled" or "always"
>

// ❌ KeyboardAvoidingView can block touches on Android
<KeyboardAvoidingView 
  behavior="padding"  // Try "height" on Android instead
>

// ❌ Pressable/TouchableOpacity inside another touchable
<TouchableOpacity>
  <Pressable onPress={handleLogin}>  // Nested touchables conflict
  </Pressable>
</TouchableOpacity>

// ❌ View with pointerEvents might block touches
<View style={styles.overlay} pointerEvents="box-only">
```

**Fix the ScrollView:**
```typescript
<ScrollView 
  keyboardShouldPersistTaps="handled"  // This is crucial!
  contentContainerStyle={{ flexGrow: 1 }}
>
```

### Fix 5: React Native Paper Specific Fix

React Native Paper's Button uses `Pressable` internally with ripple effects. On some Android devices/configurations, this can fail. Try this:

```typescript
import { Button } from 'react-native-paper';

<Button
  mode="contained"
  onPress={handleLogin}
  // Add these props to debug
  onPressIn={() => console.log('Press IN detected')}
  onPressOut={() => console.log('Press OUT detected')}
  // Force touch feedback
  android_ripple={{ color: 'rgba(255,255,255,0.3)' }}
  // Ensure it's not disabled
  disabled={false}
  // Add explicit touch area
  contentStyle={{ minHeight: 48 }}
>
  Sign In
</Button>
```

```typescript
    <View
      onStartShouldSetResponder={() => {
        console.log('[LoginScreen] Button wrapper touched');

      <Button onPress={() => console.log('BUTTON ONPRESS FIRED!')}>
        Sign In
      </Button>
    </View>
  </View>
</View>
```

**Expected output when tapping button:**
```
[LoginScreen] Root view touched
[LoginScreen] Form container touched
[LoginScreen] Button wrapper touched
BUTTON ONPRESS FIRED!
```

**If you see the first three but not the last:** React Native Paper's Button is the problem.

**If you see nothing:** GestureHandlerRootView or a parent is blocking touches.

### Step 2: Check for Absolutely Positioned Overlays

A common cause is an invisible overlay covering the button:

```typescript
// Search your LoginScreen for patterns like:

// ❌ Absolutely positioned views can block touches
<View style={{ 
  position: 'absolute', 
  top: 0, 
  left: 0, 
  right: 0, 
  bottom: 0,  // This covers everything!
  // If missing pointerEvents="none", it blocks touches
}}>

// ✅ Fix: Add pointerEvents
<View style={{ position: 'absolute', ... }} pointerEvents="none">
```
